# Бриф: панель студента (Next.js) + robomind-back

**Источник контракта HTTP:** `docs/API-STUDENT-PANEL.md` (синхронизировать с `robomind-back/docs/API-STUDENT-PANEL.md`).

## Продукт

Платформа курсов по робототехнике для школьников: регистрация (ИИН, ФИО, школа), доступ к курсам выдаёт админ, модули с видео/текстом/файлами/эфиром, после теста открывается следующий модуль, дашборд прогресса, ИИ-ассистент по модулю, сертификаты. Панель студента — только сценарии **ученика** (не админка).

## Бэкенд

- NestJS 11, PostgreSQL, TypeORM  
- `auth` — вход, регистрация  
- `app-api` — фронт студента: префикс `/app/...`  
- `admin-api` — админка (панель студента не вызывает)  
- `ai` — `/app/ai/...`  
- `device` — лимит 2 устройств на ученика  

**Базовый URL:** `{BACKEND_ORIGIN}/api/v1/...`

## Device ID

При логине ученик передаёт стабильный UUID устройства (localStorage). Третье устройство — 403. См. `DEVICE-LIMIT.md` на бэке.

## Авторизация

Сейчас возможен `x-user-id` для разработки. Цель: `Authorization: Bearer <access_token>`.

## Роль фронта

1. Онбординг / регистрация (город → район → школа)  
2. Логин с `deviceId`  
3. Курсы, модули, плеер, тесты  
4. Дашборд: прогресс, сертификаты, ИИ  
5. ИИ-чат по модулю  

Код на бэке: `app-api/*.controller.ts`, `auth.controller.ts`, `app-ai.controller.ts`.
